Workshop: Arduino

I had to some programming for the arduino workshop. On this page you'll find the codes for the assignments I had to put on the website. You will also find videos demonstrating how the codes work. Furthermore, you'll get a description of what the assignment was about and how the assignment was accomplished.

Assignment 3C

See the Pen Arduino 3C by Sergh (@Serghi0) on CodePen.

For this assignment I had to make sure that the green light blinks twice as fast as the yellow light. First of I had to build the circuit the way it is done in the handout with two LEDs, two resistors (both are 220 Ω) and four jump wires. The LED’s are connected to the pins 13 and 12. I adjusted the code so that two LED’s could be used instead of just one. I adjusted the code so that the green light would blink twice as fast. To do this I put a delay on the yellow light while there was no delay on the green light. You should be able to see this in the code included on the site. You will also find a video of the live demo included on the site.

Assignment 4C

See the Pen Arduino 4C by Sergh (@Serghi0) on CodePen.

This assignment is an extension of the last assignment. Instead of only turning the LEDs on and off, this circuit is able to set the brightness of a LED from 0 (completely off) to 255 (completely on) or anywhere in between. To accomplish this, the analogWrite() function was needed. Not all pins are suitable for this. It must be pins that are suitable for Pulse Width Modulation (PWM). These pins can behave as if it is partially on, meaning it can make the LED look like it is dimmed. That is why the pins 9 and 10 have been chosen.

For this assignment the code that was given had been extended so that one LED fades in and the other LED fades out at the same time. This extension can be seen in the code linked above. Furthermore, there is a video of a live demo linked above.

Assignment 6C

See the Pen Arduino 6C by Sergh (@Serghi0) on CodePen.

For this assignment I had to use the input of a potentiometer to fade LEDs. The point was to use an analog input to control an analog output. In an earlier assignment it was shown that the Arduino read a value between 0 and 1023 from a potentiometer while a LED needs a value between 0 and 255. This means that the code needs to be adjusted so that the sensor value gets scaled to the 0-255 value. The map() function was used to accomplish this. This function had five parameters. The first one is the value you want to map, the second one is the lower bound of the value’s current range, the third one is the upper bound of the value’s current range, the fourth one is the lower bound of the value’s target range and the firth one is the upper bound of the value’s target range.

For this assignment one LED had to fade in while the other had to fade out and vice versa. The code from assignment 4C was adjusted to accomplish this. This can be seen in the code provided above.

The circuit was built like the handout indicated. The centre lug of the potentiometer was connected to pin A0 of the Arduino. The LEDs are connected to pin 8 and 9 of the Arduino.

Assignment 7

See the Pen Assignment 7 by Sergh (@Serghi0) on CodePen.

For this assignment the objective was to dim the lights using a Light Dependent Resistor. This is a sensitive sensor that measures the amount of light. Apparently, this is similar to the sensor used to turn street lights on when it gets dark outside.

One leg of the LDR was connected to the 5V pin of the Arduino while the other leg was connected to the A0 pin of the Arduino. The leg that is connected to the A0 pin is also connected to a 10kΩ resistor which is connected to the GND pin of the Arduino. The LED pins are connected to the Arduino at the pins 10 and 9 respectively. These LEDs were connected to 220Ω resistors.

The map() function was used the same way as it was used in the previous assignment to covert the values received from the sensor to the values needed for the LEDs. The contrain() function was used to make sure the value was never lower than 0.

Assignment 9F

See the Pen Arduino 9F by Sergh (@Serghi0) on CodePen.

For this assignment I had to make use of two buttons and a LED. For this assignment the LED had to light up when one button of the two had been pressed, but not both. To accomplish this I used the function digitalWrite(). This function uses two parameters, the first one gives the pin number and the second one needs the value. In this assignment the first parameter is the LED pin.The second parameter is digitalRead(buttonPinL) != digitalRead(buttonPinR). This means that when the value of the first one button is read, the value of the second one isn’t applicable.

The circuit was built as is shown in the handout. The LED is coupled to the Arduino at pin number 8 via a jump wire. The buttons are coupled to pin number 2 and 3 of the Arduino respectively via jump wires. There are two different types of resistors used in the circuit. The resistor closest to the LED is 220Ω and the resistor closest to the button is 10kΩ. The breadboard was connected to the 5V pin and the GND pin of the Arduino and the buttons were connected to the 5V pin via the breadboard.

Assignment 10B

See the Pen Arduino 10B by Sergh (@Serghi0) on CodePen.

For this assignment I had to make use of a servomotor. Servomotors are motors that turn to a precise position. They are applicable for many purposes, such as to turn the wheel of a remote controlled car or the helm of a remote controlled boat. A servomotor does have its faults though. Most servomotors can not rotate completely. Usually they have a reach of ±180 degrees.

The circuit was built the way it was shown in the handout. The breadboard was connected to the 5V pin and the GND pin of the arduino. The servo was connected to the pin number 9 of the Arduino and the 5V pin and the GND pin via the breadboard.

In this assignment I had to make the servomotor move in interesting ways that aren’t generic like the code that was given beforehand. The adjustment to the code is shown above with a video of the live demo shown.

For this assignment I had to control the servo with inputs. I had to use two buttons to control the servo. Assignment 12B

See the Pen Arduino 12B by Sergh (@Serghi0) on CodePen.

The circuit consists of a Piezo speaker, a 220Ω resistor and four jump wires. Due to the speaker having a polarity indicated by the + sign next to one of the lugs, the + will have to go towards the Arduino’s pin 8. The resistor has no polarity, so the orientation doesn’t matter there.

To test the speaker, the function tone() is needed. This function makes the tone. It consists of the three parameters with different responsibilities. The first parameter states which pin the speaker is connected to. The second parameter determines the frequency of the tone (in hertz). The final parameter states the duration of the tone (in milliseconds). The last parameter is optional.

For this assignment I had to create a creative melody by expanding on the code given beforehand. To accomplish this, the diagram was used to choose the frequencies. I tried to make the Star Wars Imperial March theme. I failed but the melody is still nice to hear.

Assignment 13E

See the Pen MXQMeY by Sergh (@Serghi0) on CodePen.

Description

For this assignment I had to make an electronic spirit level (waterpas in Dutch). I needed to use an Inertial Measurement Unit, a combination of an accelerometer and a gyroscope. Thus, it measures the G-forces in three directions and measures the rotation giving you six values. It gives you six degrees of freedom (DOF). First, I had to calibrate the sensor to get precise values. Then I had to visualize the movement of the sensor.

For this assignment I had to change the Arduino code I received in a previous assignment so that a tone sounds as soon as the accelerometer is (almost) level. To accomplish this, I had to use the Pitch and the Roll. I used a margin of -2 to 2 degrees because if I had chosen the value 0 for the Pitch and Roll, it would hardly be measurable. In the code you can see that I made a function called checkAccelerometer() to accomplish this.
